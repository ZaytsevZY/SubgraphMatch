# 文献调研文档

## 1. 文档目的
本文件用于整理子图匹配领域的调研框架，服务于课程作业中的以下任务：
- 构建文献分类体系
- 对候选论文进行比较并确定复现目标
- 为最终报告的调研部分提供素材

## 2. 建议分类维度

### 2.1 按核心技术路线分类
- 搜索与回溯
- 过滤 + 验证
- 动态规划
- 图分解
- 索引加速
- 嵌入或编码方法

### 2.2 按问题场景分类
- 静态图子图匹配
- 动态图子图匹配
- 稠密图或超大图子图匹配
- 带标签图子图匹配
- 子图同态

### 2.3 按优化目标分类
- 缩小候选集
- 降低搜索树规模
- 减少 `partial mappings`
- 优化匹配顺序
- 提升剪枝效率
- 支持动态维护或结果复用

## 4. 论文选择建议
当前建议先重点比较以下三篇：
- `[13]`：代表较完整的现代精确匹配优化路线，但实现细节可能较多
- `[14]`：如果 guard-based pruning 拆解清晰，可能最适合课程项目做“方法 + 消融”
- `[17]`：经典且资料较多，若实现风险控制优先，可作为保底方案

## 5. 每篇论文建议记录的信息
后续阅读时，建议为每篇论文至少补充以下内容：
- 问题定义
- 核心思想
- 搜索框架
- 过滤策略或索引策略
- 匹配顺序设计
- 数据集与查询集来源
- 实验指标
- 适合复现程度评估
- 是否便于做过滤策略消融

## 6. 近期产出建议
建议在 `docs/literature-review/` 目录下尽快新增以下文件：
- `paper-selection.md`：候选论文比较和最终选择理由
- `taxonomy.md`：子图匹配文献分类体系

## 7. 当前判断
在还没有逐篇精读前，不建议立刻承诺复现最复杂的方法。对课程项目来说，最优方案通常不是“最先进”，而是“最能稳定实现 + 最容易解释 + 最容易做消融”的那一篇。
