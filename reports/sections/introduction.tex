\section{Introduction}

Exact subgraph matching is a fundamental graph query problem with applications in social-network analysis, biological interaction analysis, fraud detection, and graph database systems. Given a large vertex-labeled data graph and a smaller query graph, the goal is to enumerate all injective mappings that preserve both labels and edges. Because the search space grows rapidly with query size, practical systems rely heavily on candidate filtering, matching-order optimization, and state-aware pruning.

This paper studies \emph{GuP}, a recent exact subgraph matching method based on guard-based pruning~\cite{arai2023gup}. GuP introduces two complementary pruning ideas: reservation guards, which propagate injectivity-related constraints to earlier search steps, and nogood guards, which attempt to reuse dead-end information discovered during the search. These mechanisms are designed to reduce futile recursive exploration while preserving exactness.

The study combines a literature review, a focused reproduction of GuP, experiments on multiple benchmark datasets, an ablation of the two guard mechanisms, and a small extension on result reuse across related queries. Together, these parts make it possible to examine both the core pruning ideas of GuP and their behavior on real workloads.

\subsection{Implementation Overview}

The implementation consists of a baseline depth-first backtracking matcher and a second matcher that adds GuP-inspired reservation and nogood guards on top of the same search framework. The code is written in Python and emphasizes correctness, interpretability, and direct comparison across variants.

The remainder of the paper is organized as follows. Section~\ref{sec:lit-review} reviews the literature and positions GuP within a multi-dimensional taxonomy of exact subgraph matching methods. Sections~\ref{sec:problem-def}--\ref{sec:impl-details} define the problem, summarize GuP, and describe the implementation. Sections~\ref{sec:exp-setup}--\ref{sec:ablation} present the main experiments and the ablation study. Section~\ref{sec:optional-reuse} reports the result-reuse extension for related queries. Section~\ref{sec:conclusion} concludes the paper.
